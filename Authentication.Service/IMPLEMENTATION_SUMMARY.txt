 1️⃣  GET /api/v1/auth/public-key-info ✅
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  📌 Fichier:  AuthApiController.cs (ligne 93-107)
  
  🎯 Fonction: Retourne la configuration JWT publique
  
  🔍 Réponse:
     {
       "issuer": "GestionDesStagiaires",
       "audience": "GestionDesStagiairesApp",
       "algorithme": "HS256",
       "dureeMinutes": 60,
       "refreshDureeJours": 7,
       "clockSkewSeconds": 5
     }
  
  💡 Utilité: Les autres services appellent cet endpoint pour récupérer
             la configuration JWT nécessaire pour valider les tokens.
  
  ✅ Status: IMPLÉMENTÉ ET PRÊT

 2️⃣  POST /api/v1/auth/login ✅
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  📌 Fichier:  AuthApiController.cs (ligne 28-70)
  
  🎯 Fonction: Authentifie l'utilisateur et retourne les tokens JWT
  
  📤 Requête:
     {
       "email": "admin@system.com",
       "motDePasse": "Admin@1234"
     }
  
  📥 Réponse (200 OK):
     {
       "success": true,
       "accessToken": "eyJhbGciOiJIUzI1NiIs...",
       "refreshToken": "base64_encoded_token",
       "expiration": "2026-07-15T12:35:00Z",
       "utilisateur": {
         "id": 1,
         "nom": "Admin",
         "prenom": "Super",
         "email": "admin@system.com",
         "role": "Administrateur",
         "permissions": ["UTILISATEUR_CONSULTER", "UTILISATEUR_LISTER", ...]
       }
     }
  
  ⚠️  Erreurs:
     - 401 Unauthorized: Identifiants invalides
     - 403 Forbidden: Compte désactivé
  
  ✅ Logique Implémentée:
     ✓ Validation des identifiants (BCrypt)
     ✓ Vérification du statut du compte
     ✓ Récupération des permissions du rôle
     ✓ Génération du JWT (HMAC-SHA256)
     ✓ Génération du Refresh Token
     ✓ Sauvegarde en BDD
  
  ✅ Status: IMPLÉMENTÉ ET PRÊT

 3️⃣  POST /api/v1/auth/refresh-token ✅
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  📌 Fichier:  AuthApiController.cs (ligne 72-108)
  
  🎯 Fonction: Renouvelle le access token en utilisant le refresh token
  
  📤 Requête:
     {
       "refreshToken": "base64_encoded_token"
     }
  
  📥 Réponse (200 OK):
     {
       "success": true,
       "accessToken": "eyJhbGciOiJIUzI1NiIs...",
       "refreshToken": "new_base64_encoded_token"
     }
  
  ⚠️  Erreurs:
     - 401 Unauthorized: Refresh token invalide ou expiré
  
  ✅ Logique Implémentée:
     ✓ Validation du Refresh Token
     ✓ Vérification de l'expiration
     ✓ Vérification que le token n'est pas révoqué
     ✓ Révocation de l'ancien Refresh Token
     ✓ Génération du nouveau Access Token
     ✓ Génération du nouveau Refresh Token
     ✓ Sauvegarde en BDD
  
  ✅ Status: IMPLÉMENTÉ ET PRÊT

  📌 Fichier:  AuthApiController.cs (ligne 110-122)
  
  🎯 Fonction: Révoque le refresh token pour déconnecter l'utilisateur
  
  📤 Requête:
     {
       "refreshToken": "base64_encoded_token"
     }
  
  📥 Réponse (200 OK):
     {
       "success": true,
       "message": "Déconnecté."
     }
